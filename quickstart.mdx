## Welcome to Docket

This guide will walk you through creating your first note and using Docket's core features. By the end, you'll be productive with markdown notes right in your browser.

## Your First Note

<Steps>
  <Step title="Open Docket">
    Open a new tab in Chrome. You should see the Docket interface with a default note ready.
    
    The interface has three main areas:
    - **Sidebar** (left) - List of all your notes
    - **Editor** (center) - Where you write markdown
    - **Preview** (right) - Live rendered view of your markdown
  </Step>
  
  <Step title="Name your note">
    Click the title field at the top and give your note a meaningful name, like "My First Note" or "Quick References".
    
    <Note>
      Note titles appear in the sidebar for easy navigation. They're automatically saved as you type.
    </Note>
  </Step>
  
  <Step title="Write some markdown">
    In the editor panel, try writing some markdown:
    
    ```markdown
    # Welcome to Docket
    
    This is my first note. Markdown makes it easy to:
    
    - Create **bold** and *italic* text
    - Build lists and checklists
    - Add code snippets
    - Link to [websites](https://example.com)
    
    ## Code Example
    
    You can add code blocks with syntax highlighting for JavaScript, Python, and more!
    ```
    
    Watch the preview panel update in real-time as you type!
  </Step>
  
  <Step title="See the live preview">
    The right panel shows your rendered markdown with:
    - Formatted text and headers
    - Syntax-highlighted code blocks
    - Clickable links
    - Styled lists and quotes
  </Step>
</Steps>

## Essential Keyboard Shortcuts

Speed up your markdown editing with these keyboard shortcuts:

<AccordionGroup>
  <Accordion title="Text Formatting">
    - **Ctrl+B** - Make selected text **bold**
    - **Ctrl+I** - Make selected text *italic*
    - **~** - Wrap selected text with ~~strikethrough~~ (type ~ with text selected)
    - **`** - Wrap selected text as `inline code` (type backtick with text selected)
  </Accordion>
  
  <Accordion title="Links & Navigation">
    - **Ctrl+L** - Create a markdown link from selected text
    - **Ctrl+H** - Toggle the sidebar visibility
    - **Tab** - Insert a tab character in the editor
  </Accordion>
</AccordionGroup>

### Try it yourself

1. Type some text in the editor
2. Select the text with your mouse or keyboard
3. Press **Ctrl+B** to make it bold
4. Select different text and press **Ctrl+I** for italic

The markdown syntax is added automatically around your selection!

## Customize Your Experience

### Toggle Dark Mode

<Steps>
  <Step title="Find the theme toggle">
    Look for the dark mode slider in the sidebar settings area
  </Step>
  
  <Step title="Switch to dark mode">
    Click the toggle to enable dark mode. The entire interface switches to a dark theme that's easier on the eyes.
  </Step>
</Steps>

### Change Code Highlighting Style

Docket supports multiple syntax highlighting themes for code blocks:

<Steps>
  <Step title="Open the code style dropdown">
    Find the code style selector in the sidebar (default is "github")
  </Step>
  
  <Step title="Choose a theme">
    Select from available themes to change how code blocks are highlighted in the preview
  </Step>
</Steps>

### Switch View Modes

Docket offers three view modes to match your workflow:

<CodeGroup>
```text Editor Mode
┌─────────────┐
│             │
│   Editor    │
│    Only     │
│             │
└─────────────┘
```

```text Split Mode (default)
┌──────┬──────┐
│      │      │
│Editor│Review│
│      │      │
└──────┴──────┘
```

```text Reader Mode
┌─────────────┐
│             │
│   Preview   │
│    Only     │
│             │
└─────────────┘
```
</CodeGroup>

Use the view mode radio buttons in the toolbar to switch between:
- **Editor** - Focus on writing without distractions
- **Split** - See markdown and preview side-by-side (default)
- **Reader** - Read your rendered notes without the editor

## Working with Multiple Notes

### Create a new note

<Steps>
  <Step title="Click the new note button">
    Find the "+" or "New Note" button in the sidebar
  </Step>
  
  <Step title="Start writing">
    Your new note is created and ready. Give it a title and start adding content.
  </Step>
</Steps>

### Organize your notes

- **Reorder notes**: Drag and drop notes in the sidebar to rearrange them
- **Switch notes**: Click any note in the sidebar to open it
- **Delete notes**: Use the delete button (confirm when prompted)

<Warning>
  Deleted notes cannot be recovered. Consider exporting important notes as backups.
</Warning>

### Export a note

To save a note as a markdown file:

<Steps>
  <Step title="Open the note you want to export">
    Click the note in the sidebar to make it active
  </Step>
  
  <Step title="Click the download button">
    Find and click the download/export button in the toolbar
  </Step>
  
  <Step title="Save the file">
    Your note downloads as a `.md` file with the note's title as the filename
  </Step>
</Steps>

## Code Syntax Highlighting

Docket supports syntax highlighting for multiple programming languages:

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    const notes = await fetchNotes();
    notes.forEach(note => {
      console.log(note.title);
    });
    ```
  </Tab>
  
  <Tab title="Python">
    ```python
    def process_notes(notes):
        for note in notes:
            print(note['title'])
    ```
  </Tab>
  
  <Tab title="Rust">
    ```rust
    fn process_notes(notes: Vec<Note>) {
        for note in notes {
            println!("{}", note.title);
        }
    }
    ```
  </Tab>
  
  <Tab title="TypeScript">
    ```typescript
    interface Note {
      title: string;
      body: string;
      uuid: string;
    }
    
    function displayNote(note: Note): void {
      console.log(note.title);
    }
    ```
  </Tab>
</Tabs>

Supported languages: C, C++, Java, JavaScript, Python, Rust, TypeScript, and plaintext.

## Advanced Markdown Features

Docket uses GitHub Flavored Markdown (GFM) with additional extensions:

### Footnotes

```markdown
Here's a sentence with a footnote[^1].

[^1]: This is the footnote content.
```

### Alerts (GitHub-style)

```markdown
> [!NOTE]
> Useful information that users should know.

> [!WARNING]
> Critical content demanding immediate attention.
```

### Task Lists

```markdown
- [x] Complete the quickstart guide
- [ ] Create my project notes
- [ ] Export important notes as backup
```
